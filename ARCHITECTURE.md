# WBAnalyzer — Техническая документация

## Общая архитектура
- **Стек:** Python 3.9, один файл app.py, PostgreSQL (Railway), HTTP сервер порт 8080
- **AI:** Anthropic API (claude-sonnet-4-5)
- **Данные:** MPStats API
- **Хостинг:** Railway (git push = автодеплой)

---

## Источники данных

### MPStats API
- Endpoint: https://mpstats.io/api/wb/get/category
- Возвращает: товары ниши с полями sales, revenue, turnover_days, warehouses_count, is_fbs, balance, balance_fbs, frozen_stocks_percent, days_in_stock, avg_price, rating и др.
- **Важно:** MPStats НЕ возвращает названия конкретных складов WB — только их количество (warehouses_count)

### База данных (PostgreSQL)
- Таблица niches — 7534 ниши WB с метриками
- Поля: name, revenue, sellers, sellers_with_sales, buyout_pct, turnover, profit_pct, lost_revenue_pct, products, products_with_sales, avg_rating, display_name, mpstats_path

## Агенты (sticky панель снизу карточки ниши)

### Глубокий анализ
- Endpoint: GET /deep-analysis
- Данные: метрики ниши из MPStats (revenue, sellers, buyout_pct, turnover и др.)
- Логика: Claude анализирует данные и возвращает HTML с инсайтами, гипотезами, рекомендациями
- Модель: claude-sonnet-4-5, max_tokens=3000

### Юнит-экономика
- Endpoint: POST /unit-economy
- Данные: данные ниши + пользовательский ввод (себестоимость, схема FBO/FBS/Китай)
- Логика: Claude рассчитывает юнит-экономику с учётом комиссий WB, логистики, налогов РБ
- Сценарии: FBO (прямо на WB), FBS (со своего склада), Китай->РБ->WB

### Реклама
- Endpoint: POST /ad-analysis
- Данные: avg_cpm, ad_pct из MPStats (топ-30 SKU)
- Логика: Claude анализирует рекламную ситуацию и даёт рекомендации по бюджету
- Метрики: avg_cpm (средняя ставка), ad_pct (процент товаров с рекламой)

### Поставки (включает географию складов)
- Endpoint: POST /warehouse-analysis
- Данные: warehouse_stats из MPStats (агрегация топ-30 SKU)
- Что считаем: fbs_count, fbo_count, avg_wh_leaders, avg_wh_others, avg_turnover, frozen_pct, avg_balance, avg_balance_fbs
- Важно: MPStats не даёт названия складов — Claude рассчитывает распределение на основе статистики FBS/FBO
- Контекст РБ: приоритет складов близких к Беларуси: Смоленск (приоритет 1), Коледино, Подольск, Электросталь, Шушары
- Результат: рекомендуемая модель (FBS/FBO/Mixed), распределение по складам с цифрами, объём первой поставки

### Документы
- Endpoint: POST /docs-analysis
- Данные: название ниши, средняя цена
- Логика: Claude определяет какие сертификаты/документы нужны для товара в РБ и России

### Поставщики
- Endpoint: POST /supplier-analysis + GET /photo-supplier-search
- Данные: название ниши + опционально фото товара (Vision API)
- Логика: Claude ищет поставщиков на 1688/Alibaba/Made-in-China/AliExpress, возвращает цены
- Vision: если загружено фото — Claude анализирует его и ищет похожие товары

### Агент контента
- Endpoint: GET /content-agent
- Данные: название ниши, средняя цена
- Логика: Claude генерирует готовый контент для карточки WB
- Блоки: SEO заголовок, описание, буллеты, характеристики, рекомендации по фото, совет по видео

## Разделы платформы

### Все ниши (каталог)
- Данные из таблицы niches PostgreSQL
- Фильтры: поиск по названию, сортировка по выручке/прибыли/обороту
- История последних просмотров (localStorage)

### Топ ниши
- Алгоритм: revenue > 0, profit_pct > 0.1, buyout_pct > 0.5, sellers > 10
- Score: активность продавцов + выручка на продавца + выкуп + маржа + оборачиваемость
- Обновление: кнопка Показать другие (offset +21)

### Подбор портфеля
- AI подбор ниш под бюджет и параметры пользователя
- Endpoint: POST /portfolio-ai

### Калькулятор
- Юнит-экономика FBO/FBS/Китай с ручным вводом
- Мультивалютность: руб/$/евро/Br с реальными курсами

### В работе (watchlist)
- Хранение: localStorage (watchlist)
- Ниши отобранные для дальнейшей проработки

### Портфель
- Хранение: localStorage (portfolio)
- Товары с статусами: планируется/заказан/в пути/таможня/на складе WB/продаётся

### Компания
- Хранение: localStorage (company_settings)
- Блоки: сотрудники с налогами РБ (ФСЗН 34% + ПН 13%), склад РБ, налоговый режим, прочие расходы
- Cashflow: 24 месяца с WB вычетами (комиссия + возвраты + логистика), фазы проекта, путь к окупаемости

---

## Мультивалютность
- Базовая валюта хранения: рубли
- Курсы: загружаются с ЦБ РФ при старте
- Валюты: руб (rub=1), $ (usd~0.011), евро (eur~0.010), Br (byn~0.036)
- Конвертация: value_in_currency = value_rub * rates[currency]
- В разделе Компания: базовая валюта $, конвертация через usdRate

---

## Налоги РБ (раздел Компания)
- ФСЗН: 34% от оклада (платит работодатель)
- Подоходный налог: 13% от оклада (платит работник)
- Полная стоимость сотрудника: оклад * 1.34
- Налоговые режимы: УСН6% / УСН15% / ОСН

## Технические особенности

### localStorage ключи
- watchlist — ниши В работе
- portfolio — товары портфеля
- company_settings — настройки раздела Компания

### JS переменные
- window.currentNiche — текущая открытая ниша
- window._warehouseStats — статистика складов топ-30 SKU
- currentCurrency — текущая валюта
- rates, symbols — курсы и символы валют

### Структура app.py
- Строка 238: HTML = начало HTML/JS (один большой файл)
- Внутри HTML: один блок script с ~4000 строк JS
- После HTML: класс Handler с do_GET и do_POST
- Деплой: git add . && git commit && git push
- Откат: git checkout -- app.py

### Порядок меню
Все ниши -> Топ ниши -> Подбор -> Калькулятор -> В работе -> Портфель -> Компания
